#Loop

once it loop through the map, it will create a class for each key-value pair


@each $key, $val in $colors {
.text-#{$key} {
color: $val;
}
}


for example: .text-primary { color: #326dee; }


#{$key} is the selector interpolation


Explaination


The @each directive in SCSS is a control directive that is used to loop through lists and maps. In your code, $colors is expected to be a map where each entry has a key and a value. The @each directive is iterating over each entry in the $colors map, assigning the key of the current entry to the $key variable and the value of the current entry to the $val variable.


Then on each iteration of the @each loop, $key would be one of the color names (like "primary", "secondary", etc.) and $val would be the corresponding color value (like #007bff, #6c757d, etc.).


For Loop


@for $i from 1 through 9 {
.text-#{$key}-light-#{$i} {
color: mix(white, $val, $i * 10 * 1%);
}
.text-hover-#{$key}-light-#{$i} {
&:hover {
color: mix(white, $val, $i * 10 * 1%);
}
}
.bg-#{$key}-light-#{$i} {
background-color: mix(white, $val, $i * 10 * 1%);
}
}




Here's a breakdown of what the code does:

@for $i from 1 through 9: This is a loop that iterates 9 times. The variable $i will take on values from 1 to 9 in each iteration.

.text-#{$key}-light-#{$i}: This line generates a CSS class name by concatenating the string .text-, the value of $key, the string -light-, and the value of $i. For example, if $key is "primary", on the first iteration, the class name would be .text-primary-light-1.

color: mix(white, $val, $i * 10 * 1%);: This line sets the color of the text for the generated class. It uses the mix() function to mix white with the color stored in $val, with the degree of mixing determined by $i * 10 * 1%. As $i increases, the color becomes lighter.

.text-hover-#{$key}-light-#{$i} and &:hover: These lines generate a hover effect for the text. When you hover over the text, the color will change according to the mix() function.

.bg-#{$key}-light-#{$i}: This line generates a CSS class for setting the background color. The background color is set to a lighter variation of the color stored in $val.

This code is typically used in a larger loop that iterates over a map of color keys and values, where $key is the color name and $val is the color value. The result is a set of CSS classes that you can use to easily apply different shades of your theme colors to text and backgrounds.

# Function



@function light-comp($color) {
$complement: complement($color);
$light-complement: lighten($complement, 30%);
@return $light-complement;
}


explainiation


$complement: complement($color);: This line uses the built-in complement() function to get the complement of the input color. The complement of a color is the color directly opposite it on the color wheel, which results in a high contrast color.

$light-complement: lighten($complement, 30%);: This line uses the built-in lighten() function to lighten the complement color by 30%. The lighten() function increases the lightness of a color in the HSL color space by a certain percentage.

@return $light-complement;: This line returns the resulting light complement color. This is the value that will be output when the light-comp() function is called.

So, if you call light-comp(#0000ff), the function will calculate the complement of #0000ff (which is #ffff00), lighten that complement by 30%, and then return the resulting color.

# Purging


purging is a way to remove all the unused css classes, in order to do that you need to install a package called purgecss


use this command -> npm install gulp-purgecss --save-dev



then add "const purgecss = require('gulp-purgecss') into your gulpfile.js


in buldStyles add ".pipe(purgecss({content: ['*html']}))" -> content property tells the plugin which files to look in to determine what css rules we're actually using for our website


then run again "gulp" in the terminal

SASS (Syntactically Awesome Stylesheets) is a CSS preprocessor that provides features not available in plain CSS. Here are some key features that SASS offers:

Variables: SASS allows you to define variables that can be reused throughout your stylesheet.

Nesting: SASS lets you nest your CSS selectors in a way that follows the same visual hierarchy of your HTML.

Mixins: These are like functions for CSS. You can define a mixin once and then reuse it throughout your stylesheet, even passing in different arguments each time.

Inheritance (Extend/Inherit): SASS provides a way to share a set of CSS properties from one selector to another.

Operators: SASS supports standard math operators like +, -, *, /, and %.

Control Directives: SASS provides control directives for libraries, such as @if, @for, @each, and @while.

Importing: SASS provides a more powerful @import that helps you to split your CSS into smaller, more maintainable portions.

Functions: SASS provides many built-in functions for manipulating colors and other values.

Partials and Modules: You can create partial SASS files that contain little snippets of CSS that you can include in other SASS files.

These features make SASS a powerful tool for writing more maintainable, scalable, and DRY (Don't Repeat Yourself) CSS.
